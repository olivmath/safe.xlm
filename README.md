# Safe.xlm

**Carteiras Multisignature na Stellar Soroban**


### O Problema

Hoje, a maioria das organizações, DAOs e grupos que gerenciam fundos cripto enfrentam um dilema: 

- confiam em uma única pessoa com as chaves privadas
ou
- usam soluções complexas e caras.

Empresas perdem milhões por má gestão de chaves.
Famílias perdem heranças digitais.
Projetos Web3 ficam vulneráveis a ataques internos.

### A Solução

O **Safe.xlm** resolve isso de forma simples e acessível.


Com o Safe.xlm você:

- **Cria carteiras compartilhadas** em segundos, definindo quantas assinaturas são necessárias para aprovar transações (2 de 3, 3 de 5, etc.)
- **Gerencia transações** com transparência total - cada proposta fica visível para todos os owners
- **Integra facilmente** com qualquer carteira do ecossistema Stellar

### Por que Stellar/Soroban?

Escolhemos Stellar porque oferece suporte nativo para transações multisig além da possibilidade de extender usando Soroban.

## Tecnologias

- **Frontend:** React 19, TypeScript, Vite
- **Blockchain:** Stellar Native Transaction + Soroban Smart Contracts 
- **Wallet:** Freighter API
- **UI:** Tailwind CSS, Radix UI, Lucide Icons

## Features

- Criar carteiras multisig com N de M assinaturas
- Gerenciar múltiplas carteiras
- Propor e aprovar transações
- Tutorial interativo para novos usuários
- Suporte a dark/light mode
- Integração nativa com Freighter wallet
